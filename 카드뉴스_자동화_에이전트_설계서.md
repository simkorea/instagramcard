# 카드뉴스 자동화 에이전트 설계서

> **Claude Code 구현을 위한 계획서**
> 작성일: 2025년 | 최종 수정: 2026-04-23

---

## 작업 컨텍스트

### 배경

20~50대 부동산 관심층(아파트 매매·전세·청약·투자 관심자) 대상 부동산 뉴스 카드뉴스를 제작하는 크리에이터가,
매주 반복되는 트렌드 수집 → 주제 선정 → 스크립트 작성 → 카드뉴스 디자인 작업에 소요되는
약 11시간을 30분으로 단축하기 위한 자동화 시스템.

### 목적

키워드 입력 (또는 "이번 주 트렌드 찾아줘" 한 마디)만으로,
인스타그램 최적화 PNG 카드뉴스 6~8장을 자동 생성하고 인스타그램에 자동 업로드한다.

### 범위

**포함:**
- 인스타그램(탐색/릴스), 유튜브, 네이버(트렌드/블로그) 트렌드 수집
- 주제 후보 도출 및 사용자 선택
- 슬라이드별 스크립트 작성
- HTML 기반 카드뉴스 렌더링
- PNG 파일 변환 및 저장
- 실제 SNS 업로드 (인스타그램 캐러셀 자동 업로드 포함)
- 해시태그 자동 생성 (15~30개)
- 이미지 소재 수집 (Unsplash CDN)

**제외:**
- 캔바/피그마 연동
- 유튜브 업로드

### 입력 정의

| 입력 | 형식 | 설명 | 예시 |
|------|------|------|------|
| 키워드 | string (선택) | 콘텐츠 방향 힌트 | "트렌드 검색 아파트" |
| 자유 명령 | string (선택) | 키워드 없이 트렌드 자동 탐색 | "이번 주 트렌드 찾아줘" |

### 출력 정의

| 출력 | 형식 | 설명 |
|------|------|------|
| 카드뉴스 PNG | `.png` × 6~8장 | 1080×1350px, 인스타그램 최적화 |
| 저장 폴더 | `/output/final/YYYYMMDD_주제명/` | 카드뉴스별 날짜+주제명 폴더 |
| 스크립트 원본 | `.md` | 슬라이드별 텍스트 내용 |
| 해시태그 | `.md` | 15~30개 해시태그 |
| 인스타그램 URL | URL string | 업로드 완료 후 게시물 URL |
| 스레드 URL | URL string | 스레드 업로드 완료 후 게시물 URL |

### 제약조건

- 슬라이드 구성: **커버 1장 + 본문 4~6장 + 마무리 1장** 고정
- 이미지 사이즈: **1080 × 1350px** (세로형)
- 디자인: **모바일 퍼스트** — 한글 텍스트를 크게, 여백을 넉넉히, 가독성 중심
- 시각 요소: **사진/인포그래픽 포함**으로 시각 집중도와 전달력을 높임
- 주제 선정 단계에서 **사용자가 직접 확인/선택**
- 트렌드 채널: 인스타그램, 유튜브, 네이버
- 인스타그램 계정: **@aptshowhome**
- 대상 채널 특성: 아파트/부동산 주제 비중 높음

### 용어 정의

| 용어 | 정의 |
|------|------|
| 카드뉴스 | 인스타그램용 슬라이드 형식의 정보 콘텐츠 (6~8장) |
| 슬라이드 | 카드뉴스를 구성하는 개별 PNG 이미지 1장 |
| 스크립트 | 각 슬라이드에 들어갈 텍스트 내용 |
| 커버 | 카드뉴스 첫 번째 슬라이드 (제목 + 훅 문구) |
| 마무리 | 카드뉴스 마지막 슬라이드 (CTA + 계정 정보) |
| 인사이트 | 트렌드에서 추출한 실용적 메시지 |
| Photo Hero | 전체 배경 사진 + 다크 오버레이 레이아웃 (커버/CTA 슬라이드) |
| Photo Banner | 상단 사진 배너 + 본문 레이아웃 (본문 슬라이드) |
| SCV | 자동 실행 스크립트 (Sub-Claude Vehicle) |

## 조직도

### 조직 구조 유형

**직원 분리형** — CEO(오케스트레이터) + 전문가 4명 + SCV 4개

> 각 영역의 도메인 지식이 다르기 때문에 직원 분리 구조를 채택.

## 워크플로우

```
[입력: 키워드 or "트렌드 찾아줘"]
↓
1. SCV(trend-collector): 트렌드 수집          [SCV - 스크립트]
↓
2. 리서처: 주제 후보 도출 + 자료 리서치       [직원 - 판단]
↓
3. ⛔ 사용자 확인: 주제 선택                   [사람 개입]  ← 자동 모드만
↓
4. 콘텐츠 기획자: 슬라이드 구성 기획          [직원 - 판단] ← CEO 검토
↓
5. 카피라이터: 슬라이드별 스크립트 + 해시태그 [직원 - 판단] ← CEO 검토
↓
6. 디자이너: HTML 슬라이드 제작               [직원 - 판단] ← 자체 검증
↓
7. SCV(html-renderer): PNG 변환 및 저장       [SCV - 스크립트]
↓
8. CEO: 최종 검토                             [직원 - 판단] ← CEO 검토
↓
9. ⛔ 사용자 확인: 업로드 여부                 [사람 개입]
↓
10. SCV(instagram-uploader): 인스타그램 업로드 [SCV - 스크립트]
↓
[출력: PNG 6~8장 + 인스타그램 게시물 URL]
```

### 단계별 상세

#### 1단계 — SCV(trend-collector): 트렌드 수집
- **처리 주체**: SCV 스크립트
- **모드**: 자동 모드 (트렌드 탐색) / 키워드 모드 (직접 주제 입력)
- **수집 채널**: 인스타그램 탐색/릴스, 유튜브 급상승/쇼츠, 네이버 트렌드/블로그
- **성공 기준**: 채널별 트렌드 항목 최소 10개 수집
- **출력**: `/output/data/trend-data.json`
- **실행**: `node .claude/scv/trend-collector.js --mode auto` 또는 `--mode keyword --keyword "..."`
- **실패 시**: 자동 재시도 최대 3회 → 초과 시 CEO에게 에스컬레이션

#### 2단계 — 리서처: 자료 리서치
- **처리 주체**: 리서처 + SCV(research-crawler.js)
- **작업**: 확정 주제에 대한 사례/데이터/인사이트 수집 (3건 이상)
- **성공 기준**: 주제 관련 인사이트 3건 이상, research.md 완성
- **출력**: `/output/data/research.md`
- **실행**: `node .claude/scv/research-crawler.js --keyword "..." --output output/data/research.md`
- **실패 시**: 검색 키워드 변경 후 재시도 2회 → 에스컬레이션

#### 3단계 — 사용자 확인: 주제 선택 ⛔ (자동 모드만)
- **처리 주체**: 사람 (사용자)
- **제시 내용**: 주제 후보 3개 + 예상 카드뉴스 방향
- **사용자 액션**: 번호로 선택
- **저장**: `selectedTopic`을 `state.json`에 기록
- **키워드 모드**: 이 단계 스킵, 1단계 → 2단계 → 4단계 직행

#### 4단계 — 콘텐츠 기획자: 슬라이드 구성 기획 ← CEO 검토
- **처리 주체**: 콘텐츠 기획자
- **고정 구성**: 커버(1) + 본문(4~6) + 마무리(1)
- **성공 기준**: 전체 6~8장, 각 슬라이드 핵심 메시지 명확
- **출력**: `/output/drafts/content-plan.md`
- **실패 시**: CEO 피드백 반영 재작업 최대 2회

#### 5단계 — 카피라이터: 스크립트 + 해시태그 ← CEO 검토
- **처리 주체**: 카피라이터
- **글자 수 기준**:

| 슬라이드 | 항목 | 최대 글자 수 |
|---------|------|------------|
| 커버 | 제목 | 20자 |
| 커버 | 서브카피 | 30자 |
| 본문 | 소제목 | 20자 |
| 본문 | 내용 | 80자 |
| CTA | 문구 | 40자 |

- **해시태그**: 15~30개, `#` 포함 형식
- **출력**: `/output/drafts/script.md`, `/output/drafts/hashtags.md`
- **실패 시**: CEO 피드백 반영 재작업 최대 2회

#### 6단계 — 디자이너: HTML 슬라이드 제작 (자체 검증, CEO 게이트 없음)
- **처리 주체**: 디자이너 (또는 `/ui-ux-pro-max` 스킬 위임)
- **디자인 SSOT**: `.claude/references/design-system.md`
- **레이아웃 종류**:
  - Photo Hero: 전체 배경 사진 + 다크 오버레이 (커버/CTA — 기본, 또는 전 슬라이드 적용 가능)
  - Photo Banner: 상단 210~230px 사진 배너 + 본문 (일반 본문)
  - 인포그래픽: 다크 배경 또는 화이트 배경 + 차트/타임라인 (데이터 집약 슬라이드)
- **레이아웃 선택 기준**: 주제 성격에 따라 전 슬라이드 Photo Hero 적용도 가능 (배경 이미지가 메시지 강도를 높일 때)
- **이미지 소재**: Unsplash CDN 구 형식 ID (`photo-XXXXXXXXXXXXXXXXX`) 사용 — 신형 단문 ID 사용 불가
- **AI 이미지 생성 (선택)**: Gemini Imagen으로 커스텀 이미지 생성 가능 (`GEMINI_API_KEY` 필요)
- **스킬 위임 (선택)**: 사용자가 고급 디자인을 요청할 경우 `/ui-ux-pro-max` 스킬 Method A 적용
  - 스킬 인수에 슬라이드별 콘텐츠, 캔버스 규격, Unsplash ID 규칙, 디자인 방향을 전달
  - 각 슬라이드마다 다른 배경 이미지, 글래스모피즘 카드, 골드 강조색 적용
- **성공 기준**: design-system.md 준수, 텍스트 가독성, 슬라이드 간 통일감
- **출력**: `/output/drafts/slides/slide-1.html` ~ `slide-N.html` (하이픈 + 한 자리 숫자)
- **실패 시**: 자체 수정 최대 2회 → 미해결 시 7단계 최종 검토에서 CEO 판단

#### 7단계 — SCV(html-renderer): PNG 변환 및 저장
- **처리 주체**: SCV 스크립트 (Puppeteer)
- **저장 경로**: `/output/final/YYYYMMDD_주제명/slide-1.png` ~ `slide-N.png`
- **해상도**: 1080 × 1350px
- **성공 기준**: 모든 HTML → PNG 변환 완료, 파일 크기 정상 (100KB 이상)
- **실행**: `node .claude/scv/html-renderer.js --input output/drafts/slides/ --topic "주제명"`
- **실패 시**: 자동 재시도 최대 3회 → 초과 시 에스컬레이션

#### 8단계 — CEO: 최종 검토 ← CEO 검토
- **처리 주체**: CEO
- **검토 항목**: 슬라이드 장수(6~8), 텍스트 가독성, 브랜드 일관성, 타겟 적합성
- **성공 기준**: 전 항목 통과
- **실패 시**: 문제 단계 식별 → 해당 직원 수정 지시 (최대 2회)

#### 9단계 — 사용자 확인: 업로드 여부 ⛔
- **처리 주체**: 사람 (사용자)
- **제시 내용**: 최종 검토 통과 결과 + 업로드 여부 질문
- **다음 단계**: 승인 시 10단계 진행 / 거절 시 PNG 저장만으로 종료

#### 10단계 — SCV(instagram-uploader): 인스타그램 + 스레드 업로드
- **처리 주체**: Python 직접 실행 (instagrapi + threads-api)
- **계정**: aptshowhome (인스타그램/스레드 동일 계정, 동일 자격증명)
- **자격증명**: `.instagram_credentials.json` 참조
- **전처리**: PNG → JPEG 자동 변환 (스크립트 내부 처리)
- **JPEG 저장**: `/output/final/YYYYMMDD_주제명_jpg/`
- **세션 캐시**: `.instagram_session.json` (재로그인 방지)
- **출력**: 인스타그램 URL + 스레드 URL
- **스레드 캐러셀**: threads-api 버전에 따라 다중 이미지 미지원 시 커버(slide-1) 1장으로 자동 폴백
- **선택적 비활성화**: `--no-threads` 플래그로 스레드 업로드 건너뛰기 가능
- **이미지 공개 URL 호스팅 순서**: litterbox(catbox.moe) 우선 → 실패 시 imgur 폴백 (imgur URL은 Threads API에서 400 거부 확인됨)
- **실패 시**: 오류 메시지 출력 → 사용자 에스컬레이션

#### Unsplash 이미지 조달 절차
1. 슬라이드 주제별 구형 ID 후보 선정 (주제당 3개)
2. Node.js HTTPS HEAD 요청으로 200 확인 — 신형 단문 ID는 404 오류, 사용 불가
3. 검증된 ID로 HTML에 삽입

```bash
# ID 검증 (Node.js)
node -e "
const https=require('https');
const ids=['photo-XXXX','photo-YYYY'];
ids.forEach(id=>{
  https.request('https://images.unsplash.com/'+id+'?w=100',{method:'HEAD',timeout:6000},r=>console.log(r.statusCode+' '+id)).end();
});
"
```

**URL 형식:**
```
# Hero (전체 배경 1080×1350)
https://images.unsplash.com/photo-{구형ID}?w=1080&h=1350&fit=crop&crop=center

# Banner (상단 220px용)
https://images.unsplash.com/photo-{구형ID}?w=1080&h=440&fit=crop&crop=center
```

### 인스타그램 업로드 절차

**Step 1 — PNG → JPEG 변환:**
```python
from PIL import Image
from pathlib import Path

src = Path('output/final/YYYYMMDD_주제명')
dst = Path('output/final/YYYYMMDD_주제명_jpg')
dst.mkdir(exist_ok=True)

for png in sorted(src.glob('*.png')):
    img = Image.open(png).convert('RGB')
    img.save(dst / (png.stem + '.jpg'), 'JPEG', quality=95)
```

**Step 2 — 캐러셀 업로드:**
```python
from pathlib import Path
from instagrapi import Client

images = sorted(Path('output/final/YYYYMMDD_주제명_jpg').glob('*.jpg'))
hashtag_text = Path('output/drafts/hashtags.md').read_text(encoding='utf-8')
hashtags = [w for w in hashtag_text.split() if w.startswith('#')]
caption = '저장해두고 꺼내보세요\n\n' + ' '.join(hashtags)

cl = Client()
session_file = Path('.instagram_session.json')
if session_file.exists():
    cl.load_settings(session_file)
creds = json.loads(Path('.instagram_credentials.json').read_text())
cl.login(creds['username'], creds['password'])
cl.dump_settings(session_file)

media = cl.album_upload(paths=[str(p.resolve()) for p in images], caption=caption)
print(f'https://www.instagram.com/p/{media.code}/')
```
