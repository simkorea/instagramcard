# 카드뉴스 제작 파이프라인 에이전트 설계서

---

## 1. 작업 컨텍스트

### 배경

퇴사를 고민하거나 1인 비즈니스를 준비하는 예비 창업가들에게 인사이트를 제공하는 카드뉴스를 매일 제작하고 있다. 현재 트렌드 확인부터 스크립트 작성, 디자인, 업로드까지 수동으로 진행하며 건당 약 4~7시간이 소요된다. 이 과정을 자동화하여 15~20분(검토/수정)으로 단축한다.

### 목적

키워드를 입력하거나 "자동" 모드를 선택하면, 트렌드 기반으로 주제를 자동 추천하고, 발행 가능한 카드뉴스 이미지(PNG)와 해시태그가 자동으로 생성되는 파이프라인을 구축한다.

### 범위

- **포함**: 트렌드 수집, 주제 기획, 자료 리서치, 스크립트 작성, 카드뉴스 이미지(PNG) 생성, 해시태그 생성
- **제외**: SNS 자동 업로드(수동), 댓글/DM 응대, 유료 광고 집행

### 입력 정의

| 입력 | 형식 | 설명 | 예시 |
|------|------|------|------|
| 모드 | string | "키워드" 또는 "자동" | "자동" |
| 주제 키워드 | string (키워드 모드 시) | 카드뉴스 주제 또는 관심 분야 | "퇴사 후 첫 3개월 생존기" |

- **키워드 모드**: 사용자가 직접 주제를 입력하면 해당 주제로 리서치 후 제작 진행
- **자동 모드**: 트렌드 기반으로 주제 3개를 자동 추천 → 사용자가 선택 → 제작 진행

### 출력 정의

| 출력 | 형식 | 설명 |
|------|------|------|
| 카드뉴스 이미지 | PNG (1080×1350px) × 6~8장 | 인스타그램 세로형 규격, 깔끔/미니멀 스타일 |
| 스크립트 | .md | 슬라이드별 텍스트 전문 (검토/수정용) |
| 해시태그 | .md | 인스타그램 + 유튜브 커뮤니티용 해시태그 세트 |

### 제약조건

- 발행 빈도: 매일 1건
- 슬라이드 수: 6~8장/건
- 디자인: 깔끔/미니멀 스타일
- 이미지 규격: 1080×1350px PNG (인스타그램 세로형 4:5 비율)
- 톤앤매너: 밝고 경쾌한 톤, 말투는 주제에 따라 유동적
- 발행 채널: 인스타그램, 유튜브 커뮤니티
- 타겟 독자: 퇴사 희망자, 1인 비즈니스 예비 창업가

### 용어 정의

| 용어 | 정의 |
|------|------|
| 카드뉴스 | 슬라이드 형태의 이미지 콘텐츠. 6~8장의 세로형(4:5) 이미지로 구성 |
| 스크립트 | 카드뉴스 각 슬라이드에 들어갈 텍스트 전문 |
| 커버 슬라이드 | 첫 번째 슬라이드. 주제를 한눈에 보여주는 제목+서브카피 |
| CTA 슬라이드 | 마지막 슬라이드. 팔로우/저장/공유 유도 문구 포함 |
| 트렌드 데이터 | 타겟 독자층에서 현재 관심이 높은 주제/키워드 정보 |

## 2. 조직도

### 조직 구조: 직원 분리형

카드뉴스 제작은 리서치 → 기획 → 작성 → 디자인이라는 서로 다른 전문성이 필요한 단계로 구성되어 있어 직원 분리 구조를 채택한다.

### 직원 목록

| 직원 | 유형 | 역할 요약 |
|------|------|----------|
| **CEO** | C-Level | 전체 파이프라인 오케스트레이션, 최종 검토/승인 |
| **리서처** | 전문가 | 트렌드 수집, 자료/사례 리서치, 팩트 검증 |
| **콘텐츠 기획자** | 전문가 | 주제 선정, 슬라이드 구조 설계, 스토리라인 기획 |
| **카피라이터** | 전문가 | 슬라이드별 텍스트 작성, 해시태그 생성 |
| **디자이너** | 전문가 | 카드뉴스 비주얼 설계, HTML→PNG 변환 지시 |

### 보고 체계

```
CEO (최종 검토/승인)
├── 리서처
│    - 트렌드 데이터, 리서치 자료 보고
├── 콘텐츠 기획자
│    - 주제 추천안, 슬라이드 구조안 보고
├── 카피라이터
│    - 스크립트 초안, 해시태그 보고
└── 디자이너
     - 카드뉴스 디자인(HTML) 보고
```

### 직원 상세 정의

#### CEO

| 항목 | 내용 |
|------|------|
| 역할 | 전체 파이프라인 오케스트레이션 및 최종 품질 관리 |
| 책임 | 최종 카드뉴스 산출물 품질 승인 |
| 권한 | 주제 최종 선택, 스크립트 수정 지시, 디자인 수정 지시, 발행 승인/반려 |
| 트리거 조건 | 파이프라인 시작 시(주제 입력), 각 단계 산출물 완료 시 |
| 입력 | 사용자 주제 키워드, 각 직원의 산출물 |
| 출력 | 승인된 최종 카드뉴스 이미지 + 해시태그 |
| 보고 대상 | 사용자 |
| 참조 자료 | /references/brand-guide.md |

#### 리서처

| 항목 | 내용 |
|------|------|
| 역할 | 타겟 독자층의 트렌드 파악 및 주제 관련 자료 수집 |
| 책임 | 트렌드 데이터, 리서치 자료 품질 |
| 권한 | 검색 키워드 결정, 자료 신뢰도 판단, 핵심 인사이트 추출 |
| 트리거 조건 | CEO가 파이프라인 시작 지시 시 |
| 입력 | 주제 키워드 또는 "자동"(트렌드 기반 주제 추천 모드) |
| 출력 | /output/data/trend-data.json, /output/data/research.md |
| 보고 대상 | CEO |
| 참조 자료 | /references/target-audience.md |

#### 콘텐츠 기획자

| 항목 | 내용 |
|------|------|
| 역할 | 카드뉴스 주제 확정, 슬라이드 구조 설계, 스토리라인 기획 |
| 책임 | 기획서(슬라이드 구조안) 품질 |
| 권한 | 주제 방향 결정, 슬라이드 수 결정(6~8장), 각 슬라이드의 핵심 메시지 결정 |
| 트리거 조건 | 리서치 자료 완료 시 |
| 입력 | /output/data/trend-data.json, /output/data/research.md |
| 출력 | /output/drafts/content-plan.md |
| 보고 대상 | CEO |
| 참조 자료 | /references/brand-guide.md, /references/slide-templates.md |

#### 카피라이터

| 항목 | 내용 |
|------|------|
| 역할 | 슬라이드별 텍스트 작성 및 해시태그 생성 |
| 책임 | 스크립트 품질, 해시태그 적합성 |
| 권한 | 문구 표현 결정, 말투 선택(주제에 따라 유동적), CTA 문구 결정 |
| 트리거 조건 | 기획서 승인 시 |
| 입력 | /output/drafts/content-plan.md |
| 출력 | /output/drafts/script.md, /output/drafts/hashtags.md |
| 보고 대상 | CEO |
| 참조 자료 | /references/brand-guide.md, /references/tone-guide.md |

#### 디자이너

| 항목 | 내용 |
|------|------|
| 역할 | 카드뉴스 비주얼 설계 (HTML/CSS 기반) |
| 책임 | 디자인 품질, 브랜드 일관성, 가독성 |
| 권한 | 색상 배합 결정, 레이아웃 결정, 타이포그래피 결정 (미니멀 스타일 범위 내) |
| 트리거 조건 | 스크립트 승인 시 |
| 입력 | /output/drafts/script.md |
| 출력 | /output/drafts/slides.html (슬라이드별 HTML 파일) |
| 보고 대상 | CEO |
| 참조 자료 | /references/design-system.md, /references/brand-guide.md |

### SCV 목록

| SCV | 역할 | 호출 주체 |
|-----|------|----------|
| **trend-collector** | 트렌드 키워드/데이터 수집 (웹 검색, API 호출) | 리서처 |
| **research-crawler** | 주제 관련 자료/사례 수집 및 정리 | 리서처 |
| **html-renderer** | HTML/CSS → PNG 이미지 변환 (1080×1080px) | 디자이너 |
| **file-handler** | 파일 읽기/쓰기, 폴더 관리, 산출물 정리 | 전 직원 공용 |

```

    ├─ 키워드 모드 ──────────────────┐
    │                               ▼
    ├─ 자동 모드 ─┐    ┌─────────────────────────────────────┐
    │             ▼    │ 2단계: 자료 리서치                    │
    │  ┌──────────────────────────────────────┐              │
    │  │ 1단계: 트렌드 수집 + 주제 추천         │              │

    │  │ 처리: 트렌드 분석 → 주제 3개 추천       │              │

              ▼
```

### 단계별 상세

#### 1단계: 트렌드 수집 + 주제 추천

| 항목 | 내용 |
|------|------|
| 담당 | 리서처 + SCV(trend-collector) |
| 처리 주체 | SCV가 데이터 수집 → 리서처가 핵심 트렌드 분석/선별 |
| 입력 | 주제 키워드 또는 "자동" |
| 출력 | /output/data/trend-data.json (자동 모드 시: 추천 주제 3개 포함) |
| 성공 기준 | 최소 3개 이상의 트렌드 키워드 + 근거 데이터 포함. 자동 모드 시 추천 주제 3개 + 각 주제의 선정 이유 |
| 검증 방법 | 리서처 자체 판단 (데이터 신뢰도, 타겟 적합성) |
| 분기 | 자동 모드 → CEO(사용자)에게 주제 3개 제시 → 선택 후 2단계 진행. 키워드 모드 → 입력 키워드 특화 트렌드 수집 후 2단계 진행 |
| 실패 시 | 자동 재시도 (최대 3회) → 사용자 에스컬레이션 (오류 보고 + 수동 입력/재시도/중단 선택지 제공) |

> [구체화됨] 키워드 모드에서 1단계를 스킵하지 않고, 입력 키워드에 특화된 트렌드 수집을 실행한다.
> [구체화됨] SCV 3회 실패 시 CEO 자체 처리가 아닌 사용자 에스컬레이션으로 처리한다.

#### 2단계: 자료 리서치

| 항목 | 내용 |
|------|------|
| 담당 | 리서처 + SCV(research-crawler) |
| 처리 주체 | SCV가 자료 수집 → 리서처가 핵심 인사이트 추출/정리 |
| 입력 | /output/data/trend-data.json, 주제 키워드 |
| 출력 | /output/data/research.md |
| 성공 기준 | 주제 관련 사례/데이터/인사이트 최소 3건 이상 |
| 검증 방법 | 리서처 자체 판단 (사실 정확성, 인사이트 가치) |
| 실패 시 | 검색 키워드 변경 후 재시도 (최대 2회) → 에스컬레이션(CEO) |

#### 3단계: 주제 기획

| 항목 | 내용 |
|------|------|
| 담당 | 콘텐츠 기획자 [직원 - 판단] |
| 처리 주체 | 직원이 직접 판단 (창의적 기획) |
| 입력 | /output/data/trend-data.json, /output/data/research.md |
| 출력 | /output/drafts/content-plan.md |
| 산출물 구조 | 주제 제목, 타겟 독자, 핵심 메시지, 슬라이드 구조(6~8장), 각 슬라이드 핵심 포인트 |
| 성공 기준 | 타겟 독자 적합성, 6~8장 슬라이드 구조 완성, 각 슬라이드 핵심 메시지 명확 |
| 검증 방법 | CEO 검토 (기획 방향성, 타겟 적합성) |
| 실패 시 | CEO 피드백 반영하여 재작업 (최대 2회) |

#### 4단계: 스크립트 작성

| 항목 | 내용 |
|------|------|
| 담당 | 카피라이터 [직원 - 판단] |
| 처리 주체 | 직원이 직접 판단 (창작) |
| 입력 | /output/drafts/content-plan.md |
| 출력 | /output/drafts/script.md, /output/drafts/hashtags.md |

| 항목 | 내용 |
|------|------|
| 담당 | Python 직접 실행 (instagrapi) |
| 트리거 | 7단계 통과 후 사용자가 업로드 요청 시 |
| 계정 | aptshowhome |
| 자격증명 | `.instagram_credentials.json` 참조 (git 커밋 금지) |
| 세션 캐시 | `.instagram_session.json` — 재로그인 방지 |
| 입력 | /output/final/YYYYMMDD_주제슬러그/*.png, /output/drafts/hashtags.md |
| 출력 | 인스타그램 게시물 URL (state.json에 저장) |
| PNG→JPEG 변환 | **필수** — `album_upload`는 PNG 직접 처리 불가. PIL로 JPEG 변환 후 업로드 |
| 변환 폴더 | /output/final/YYYYMMDD_주제슬러그_jpg/ |
| 성공 기준 | 캐러셀 업로드 완료, 게시물 URL 반환 |
| 실패 시 | 세션 파일 삭제 후 재로그인 → 재시도 1회 → 사용자 에스컬레이션 |

**실행 절차 요약:**
```python
# Step 1 — PNG → JPEG
from PIL import Image
from pathlib import Path
src = Path('output/final/YYYYMMDD_주제')
dst = Path('output/final/YYYYMMDD_주제_jpg')
dst.mkdir(exist_ok=True)
for png in sorted(src.glob('*.png')):
    Image.open(png).convert('RGB').save(dst / (png.stem + '.jpg'), 'JPEG', quality=95)

# Step 2 — 캐러셀 업로드
from instagrapi import Client
import json
cl = Client()
creds = json.loads(Path('.instagram_credentials.json').read_text(encoding='utf-8'))
session_file = Path('.instagram_session.json')
if session_file.exists(): cl.load_settings(session_file)
cl.login(creds['username'], creds['password'])
cl.dump_settings(session_file)
images = sorted(dst.glob('*.jpg'))
hashtags = [w for w in Path('output/drafts/hashtags.md').read_text(encoding='utf-8').split() if w.startswith('#')]
media = cl.album_upload(paths=[str(p.resolve()) for p in images], caption='저장해두고 꺼내보세요\n\n' + ' '.join(hashtags))
print(f'https://www.instagram.com/p/{media.code}/')
```

## 부록: 슬라이드 기본 구조

카드뉴스 6~8장의 기본 슬라이드 구조:

| 슬라이드 | 역할 | 내용 |
|---------|------|------|
| 1장 (커버) | 시선 끌기 | 주제 제목 + 서브카피 (궁금증 유발) |
| 2장 | 문제 제기 / 공감 | 타겟 독자의 고민이나 상황 공감 |
| 3~5장 (또는 3~6장) | 핵심 인사이트 | 본문 내용 (포인트별 1장씩) |
| 끝에서 2번째 | 요약 / 정리 | 핵심 메시지 정리 또는 실행 가이드 |
| 마지막 (CTA) | 행동 유도 | 팔로우/저장/공유 유도 + 계정 소개 |

> 주제에 따라 6장, 7장, 8장으로 유동적으로 운영한다. 핵심 인사이트 슬라이드 수를 조절하여 전체 장수를 결정한다.

## 부록 B: state.json 스펙

> [추가됨] 세션 재개를 위한 파이프라인 상태 파일 스펙.

```json
{
  "currentStage": 8,
  "completedStages": [1, 2, 3, 4, 5, 6, 7, 8],
  "mode": "keyword",
  "selectedTopic": "4월 놓치면 안 될 청약 단지",
  "outputFolder": "output/final/20260415_4월청약단지",
  "instagramUrl": "https://www.instagram.com/p/DXJQCtrE7wL/",
  "startedAt": "2026-04-15T09:00:00+09:00",
  "updatedAt": "2026-04-15T10:30:00+09:00"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| currentStage | number | 현재 진행 중인 단계 (1~8) |
| completedStages | number[] | 완료된 단계 목록 |
| mode | string | "auto" 또는 "keyword" |
| selectedTopic | string | 선택된 주제 (3단계 이후 설정됨) |
| outputFolder | string | PNG 저장 폴더 경로 (6단계 완료 후 설정) |
| instagramUrl | string | 업로드된 게시물 URL (8단계 완료 후 설정) |
| startedAt | ISO 8601 | 파이프라인 시작 시각 |
| updatedAt | ISO 8601 | 마지막 업데이트 시각 |

CEO는 시작 시 state.json이 존재하면 `currentStage`를 읽어 해당 단계부터 재개한다. 파일이 없으면 1단계부터 시작한다.
